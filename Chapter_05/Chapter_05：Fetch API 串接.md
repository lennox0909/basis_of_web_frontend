# **第五課：與後端溝通 (Fetch API) — 數據聯網**
* HTML 部分現在包含了一個 `container` 容器，用於存放個人資料、金句展示區以及控制按鈕。
* CSS 檔案現在包含了兩套配色方案（淺色與深色），並使用了 `transition` 讓模式切換時顏色過渡更平滑。
* script.js 使用了 `async/await` 語法，這比傳統的 `.then()` 回呼函數更易讀。
   * 非同步程式 (Asynchronous)：`fetch` 是耗時的操作，使用 `await` 可以讓程式碼停在該行直到獲取結果，而不會卡死整個網頁。
   * API (應用程式介面)：這就像是向遠端的圖書館（伺服器）借書，你發出請求，它回傳 JSON 格式的資料給你。
   * Try...Catch：處理網路錯誤的標準做法，確保即使 API 掛了，網頁也不會壞掉。



## **1\. 核心概念**

讓網頁具備抓取雲端資料的能力。

## **2\. Async / Await**

網路請求需要時間等待，使用非同步語法確保程式不會在等待時「卡死」：

* async：標記函式為非同步。  
* await：暫停執行，直到資料傳回。

## **3\. Fetch API**

* fetch(URL)：向該 URL 發出請求。  
* .json()：將原始資料包轉譯為 JS 物件。

## **4\. 流程設計**

1. 發出請求 \-\> 2\. 顯示載入中 \-\> 3\. 成功拿到資料 \-\> 4\. 渲染到 HTML \-\> 5\. 錯誤處理 (Error Handling)。
